# AGENTS.md

本项目为数学建模校内赛三人协作项目（A题：扰流条件下的超声波流量计）。

## 项目结构

```
A题/
├── analysis/    # 建模手 — 分析报告、公式推导
├── code/        # 编程手 — 求解代码
├── paper/       # 论文手 — LaTeX论文
├── problem/     # 原始赛题材料（只读）
└── output/      # 中间结果、图表
```

以下规则适用于本项目所有AI辅助产出（代码注释、文档、论文片段），无需每次重复说明。

### 注释

- 只写必要的：函数一行docstring、非显然公式标注编号、关键参数取值原因
- 不写废话：`x = x + 1  # 将x加1` 这类注释禁止
- 不写过度解释：方法原理、为什么选这个方法留给论文，代码只标注做了什么

### 禁止的AI味表达

禁止以下词汇和句式出现在任何产出中：

禁用词：此外、另外、值得注意的是、深入探讨、充分展示、具有重要意义、为…奠定基础、关键的、核心的、标志着、体现了、反映了、令人振奋、前景广阔
禁用句式："不仅…而且…""从X到Y，从A到B"、"专家认为"、"观察者指出"
禁止内容：emoji表情符号、破折号过度使用、粗体强调、垂直列表式标题
禁止行为：对推导或方法自我评价——"推导全程精确""无需近似""形式优美"这类话一律不写。公式正确与否由读者判断，不需要在旁边夸它。

### 正确风格

用数据说话，不堆形容词。例：
- 正确：`模型在测试集上 MAE 0.18%，最大误差 0.50%`
- 错误：`该模型取得了优异的表现，充分证明了方法的有效性`

### 代码

- Python 3.12，依赖见 requirements.txt
- 文件头三行：`"""问题X：<一句话>。求解方法：<算法>。输出：<路径>。"""`
- 命名：snake_case 变量/函数，物理量带单位（`flow_m3h`，`length_mm`）
- 函数 ≤30行，单一职责
- 数据路径用相对路径，`problem/` 下文件只读
- Git 提交信息：`model: / solver: / paper: / fix: / chore:` + 中文简述

### 论文

- LaTeX (ctexart)，编译器 xelatex
- 用具体数据支撑每个结论，不用"效果较好""性能优异"等模糊词
- 禁止第一人称（"我们""本研究"→"本文"或省略）
- 多段叙述少分点，除非是模型假设、算法步骤
