# 问题4：达标的最终模型 — 建模分析

## 一、问题分析

### 1.1 问题重述

综合问题1–3成果，建立满足五条工程要求的最终流量估计模型：

| # | 工程要求 |
|---|---------|
| ① | 输出每个窗口的 `model_volume_m3` |
| ② | 规则和参数在测试前固定 |
| ③ | 不用 date、window_id、standard_volume 等非运行时信息 |
| ④ | 数据有效性门须有物理含义 |
| ⑤ | 报告 MAE、组通过率、$u_{\text{nor},L}$、$u_{\text{nor},r}$、$u_{\text{nor},d}$ |

### 1.2 达标目标

$$|\bar{e}_g| \le 0.2\%,\quad SD_g \le 0.040\%,\quad u_{\text{nor},L} < 0.036\%,\quad u_{\text{nor},r} < 0.040\%,\quad u_{\text{nor},d} < 0.115\%$$

### 1.3 设计原则

1. **级联架构**：问题1基线 → 问题2流量点修正 → 问题3扰流检测与补偿
2. **物理优先**：每级修正有明确的物理对应，避免纯数据驱动
3. **参数尽量少**：自由参数 ≤ 15 个，全部离线固定
4. **开关可控**：每级可独立启用/关闭，便于误差溯源和消融实验

## 二、最终模型架构

### 2.1 总流程图

```
输入：窗口在线特征
  chord0-4, profile_*, dyn_*, zero_*, flow_point
│
├─(1) 数据有效性门 ───────────────────────────
│   检查：dyn_active_eq_s ≥ 30s
│         dyn_plateau_cv ≤ 0.10
│   不通过 → 标记为无效窗口，不输出
│
├─(2) 基线估计 ──────────────────────────────
│   V_base = A · T · Σ(w_j^Lagrange · chord_j)
│   权重来自附件7，固定
│
├─(3) 流量点修正 ────────────────────────────
│   V_1 = V_base · (1 + β_p)
│   β_p 由 D0 查表，5 个固定值
│
├─(4) 扰流检测 ──────────────────────────────
│   |profile_ab_abs| > 0.035 or
│   |profile_swirl|  > 0.021
│   无扰流 → 跳到步骤(6)
│
├─(5) 扰流分类 + 补偿 ───────────────────────
│   扰流类型 c ← Mahalanobis k-NN(聚类中心)
│   V_2 = V_1 · (1 + δ_p^(c))
│
├─(6) 冷启动处理 ────────────────────────────
│   若 zero_age_s > 3600s（1小时）：
│     zero_rate_med 改用全局默认值
│     （不影响本模型——仅当模型未来扩展使用
│      零点特征时才需此分支）
│
├─(7) 输出 ──────────────────────────────────
│   model_volume_m3 = V_final
│
```

### 2.2 模块公式

#### (1) 数据有效性门

**物理含义**：窗口须包含足够稳定积分时长，且平台段流速波动可控。

| 门条件 | 阈值 | 物理推导 |
|--------|------|---------|
| `dyn_active_eq_s ≥ 30` s | 窗口最短约 116s，取 25% | 若稳定段不足窗口 1/4，则该窗口大部分时间处于过渡态，积分不可靠 |
| `dyn_plateau_cv ≤ 0.10` | 10% | 平台段 CV > 10% 意味着窗口从未真正稳定（取 ISO 12242 流量计重复性要求的保守上界） |

**不通过处理**：该窗口 `model_volume_m3` 输出 NaN，提交 CSV 中移除该行。论文中应报告被过滤的窗口数量和 ID。

#### (2) 基线积分

$$V_{\text{base}} = A \cdot T \cdot \sum_{j=0}^{4} w_j^{(L)} \cdot c_j$$

其中 $A = 0.13138$ m²，$w_j^{(L)}$ 为附件7的 Lagrange 权重（和=1.0，均值误差-0.008%），$c_j$ 为 chord0–chord4。

#### (3) 流量点修正

$$\beta_p = \frac{1}{N_p^{(0)}}\sum_{i \in \Omega_p^{(0)}}\frac{V_i^{\text{std}} - V_i^{\text{base}}}{V_i^{\text{base}}},\quad p \in \{40,50,60,70,80\}$$

$\beta_{20}$ 和 $\beta_{30}$ 用问题2的混合外推策略（线性外推 + 保守估计取平均）。

#### (4) 扰流检测

记 $p_{\text{abs}}$ = `profile_ab_abs`，$p_{\text{swirl}}$ = `profile_swirl`：

$$\text{flag} = \mathbf{1}\bigl[\,|p_{\text{abs}}| > 0.035 \;\text{OR}\; |p_{\text{swirl}}| > 0.021 \,\bigr]$$

$$\tau_1 = 0.035,\quad \tau_2 = 0.021 \qquad (\text{D0 max} + 3\sigma)$$

#### (5) 扰流补偿

扰流类型 $c$ 由 Mahalanobis 距离到离线确定的聚类中心判定（问题3子问题(2)）。补偿为加性形式：

$$\hat{V}_i = V_i^{\text{base}} \cdot \bigl(1 + \beta_{p_i} + \delta_{p_i}^{(c_i)}\bigr)$$

其中 $\delta_p^{(c)}$ 查表（≤4 类 × 7 流量点 = ≤28 个值，按问题3的 ANOVA 结果可能更少）。

#### (6) 冷启动处理

`zero_age_s` 过大时零点信息不可靠。当前模型不直接使用零点特征做补偿，但若后续迭代中加入零点修正，需备此逻辑。

## 三、参数清单与分类

### 3.1 自由参数 vs 派生常数

| 类别 | 参数 | 数量 | 定义 |
|------|------|------|------|
| **物理常数** | $A$, $w_j^{(L)}$ | 6 | 附件7固定值，非数据驱动 |
| **检验阈值** | $\tau_1$, $\tau_2$, $\tau_{\text{active}}$, $\tau_{\text{cv}}$ | 4 | 由 D0 分布 + 工程经验确定 |
| **自由参数** | $\beta_p$（5个流量点） | 5 | D0 直接查表，无迭代 |
| **派生常数** | 聚类中心（≤4×PC维数） | ≤16 | 由 D1–D8 层次聚类确定 |
| **派生常数** | $\delta_p^{(c)}$（≤4×7） | ≤28 | 由 D1–D8 组均值偏差确定 |
| **自由参数合计** | | **5** | 仅 $\beta_p$ 是真正从数据估计的参数 |

与初版 ≤47 不同，此处把聚类中心和补偿表归为"派生常数"——它们虽来自数据（D1–D8），但一旦聚类结构和分组确定，数值即固定，不属于传统意义上的可调参数。论文中应明确这一区分。

### 3.2 防过拟合措施

1. $\beta_p$ 仅从 D0 (10窗口) 估计——样本极小但参数维度极低
2. 聚类数由轮廓系数确定，上限 4 类
3. $\delta_p^{(c)}$ 按组聚合估计（同类×同流量点的所有窗口取均值），不对单窗口调优
4. 逐日留出交叉验证评估泛化性

## 四、消融实验设计

### 4.1 消融顺序

| 实验 | 配置 | 预期 MAE | 验证贡献 |
|------|------|---------|---------|
| A0 | 裸 Lagrange（问题1基线） | 0.184% | 基准 |
| A1 | A0 + β_p 流量点修正 | ↓ | 量化问题2贡献 |
| A2 | A1 + 扰流检测 + 补偿 | ↓↓ | 量化问题3贡献 |
| A3 | A2 + 有效性门 | A2 ± 微小 | 验证门不引入偏差 |
| A4 | A2 用 Phys6 替代 Lagrange | ? | 验证基线选择影响 |

### 4.2 消融评估维度

每个消融实验报告：全窗口 MAE、组通过率、$u_{\text{nor},L}$、$u_{\text{nor},r}$、$u_{\text{nor},d}$。

**关键推论**：
- 若 A1 vs A0 的 $u_{\text{nor},L}$ 显著下降 → 流量点修正有效
- 若 A2 vs A1 的 $u_{\text{nor},d}$ 显著下降 → 扰流补偿有效
- 若 A3 vs A2 无显著差异 → 有效性门未错误过滤窗口；若过滤了窗口且指标改善 → 门识别了劣质窗口

## 五、达标风险评估与备选方案

### 5.1 风险矩阵

| 指标 | 目标 | 最大风险来源 | 预期达标概率 |
|------|------|-------------|------------|
| 组均值误差 ≤ 0.2% | $\vert\bar{e}_g\vert \le 0.2\%$ | D0 β_p 外推不准确 | 高（β_p 直接消除系统偏差） |
| 组重复性 ≤ 0.040% | $SD_g \le 0.040\%$ | 扰流下窗口间自然波动 | 中高 |
| 线性度 < 0.036% | $u_{\text{nor},L} < 0.036\%$ | D0 仅5个点，每点1–3窗口 | 中（D0 样本极度有限） |
| 重复性 < 0.040% | $u_{\text{nor},r} < 0.040\%$ | 某组窗口偶然波动 | 中高 |
| 扰流指标 < 0.115% | $u_{\text{nor},d} < 0.115\%$ | 某些扰流状态补偿不足 | 中（需聚类+补偿有效） |

### 5.2 未达标时的备选方案

| 超标指标 | 诊断步骤 | 备选方案 |
|---------|---------|---------|
| $u_{\text{nor},L}$ | 检查 D0 各流量点残差是否系统非零 | 若 β_p 已消除均值但线性度仍超标 → D0 窗口间波动贡献大 → 已无改进空间（数据限制） |
| $u_{\text{nor},r}$ | 定位 $SD_g$ 最大的那个组 | 检查该组窗口的 dyn_plateau_cv 是否异常 → 收紧有效性门 |
| $u_{\text{nor},d}$ | 定位 $\vert\Delta e_{r,p}\vert$ 最大的扰流×流量点 | 将该组合从全局补偿升级为独立补偿（增加参数） |
| 多指标同时超标 | 可能是某些特定日期的系统性偏移 | 分析日期效应——若确为日期依赖（如仪器漂移），论文中建议补充更长时间跨度的 D0 数据 |

### 5.3 最难达标场景预测

- **FP=20, 30**：D0 无数据，补偿依赖外推 → $u_{\text{nor},d}$ 可能超标
- **D7**（13窗口）、**D6**（14窗口）：样本最少 → 补偿参数估计噪声大
- **D3**（2天跨日）：若跨日特征不一致 → 聚类可能错误合并或分裂

## 六、伪代码

```python
# ========== 离线阶段：一次性参数估计 ==========

# 0. 加载与基线
df = load_data()
V_base = area * df.duration_s * sum(w_L[j] * df[f'chord{j}'])

# 1. 有效性门阈值
tau_active = 30.0          # 秒, 工程经验
tau_cv = 0.10              # 工程经验

# 2. 流量点修正 β_p
D0 = df[df.condition_note == 'no_disturbance_reference']
for p in [40,50,60,70,80]:
    mask = D0.flow_point == p
    beta[p] = ((D0.standard_volume_m3[mask] - V_base[mask])
               / V_base[mask]).mean()
beta[20] = 0.5 * (extrapolate_linear(beta, 20) + min(beta[40], beta[50]))
beta[30] = 0.5 * (extrapolate_linear(beta, 30) + beta[40])

# 3. 扰流检测阈值
tau_1 = D0.profile_ab_abs.abs().max() + 3 * D0.profile_ab_abs.std()
tau_2 = D0.profile_swirl.abs().max() + 3 * D0.profile_swirl.std()

# 4. 扰流聚类（离线，可用 disturbance_id）
dist = df[df.condition_note == 'disturbed_test']
# Z-score 标准化（以 D0 为参考）
feat_matrix = build_features(dist)  # 10-dim
feat_scaled = (feat_matrix - D0_mean) / D0_std
# PCA 降维
pca = PCA(n_components=0.90).fit(feat_scaled)
feat_pca = pca.transform(feat_scaled)
# 层次聚类
by_dist = feat_pca.groupby(dist.disturbance_id).mean()  # 8×k
cluster_labels = WardClustering(by_dist).fit_predict(k=select_by_silhouette(by_dist))

# 5. 扰流补偿表 δ_p^(c)
for c in unique(cluster_labels):
    mask_c = cluster_labels[dist.disturbance_id] == c
    for p in [20,30,40,50,60,70,80]:
        mask = mask_c & (dist.flow_point == p)
        if mask.sum() >= 3:
            delta[c][p] = ((dist.standard_volume_m3[mask] - V_base[mask])
                           / V_base[mask]).mean() - beta[p]
        else:
            delta[c][p] = interpolate_or_default(delta, c, p)


# ========== 在线阶段：逐窗口推理 ==========

def predict(window):
    # 有效性门
    if (window.dyn_active_eq_s < 30 or
        window.dyn_plateau_cv > 0.10):
        return None

    # 基线 + 流量点修正
    V = area * window.duration_s * sum(w_L[j] * window[f'chord{j}'])
    V *= (1.0 + beta[window.flow_point])

    # 扰流检测 + 补偿
    if (abs(window.profile_ab_abs) > 0.035 or
        abs(window.profile_swirl) > 0.021):
        feat = extract_pca_features(window, pca, D0_mean, D0_std)
        c = nearest_cluster(feat, centers, covs)  # Mahalanobis
        V *= (1.0 + delta[c][window.flow_point])

    return V
```

## 九、实现要求

### 8.1 主要功能模块

1. **离线参数估计**：一次性计算所有固定参数，导出 `params.json`
2. **在线推理**：逐窗口执行预测，不依赖离线标签

5. **消融实验**：按 4.1 节顺序依次运行并对比

## 九、注意事项

1. 最终模型 **不得** 使用 `standard_volume_m3`、`date`、`window_id` 作为输入
2. 所有阈值和参数在测试前固定，`params.json` 与提交 CSV 一同交付
3. 有效性门的物理推导须参考 ISO 12242:2012 在论文中给出
4. 论文中区分"自由参数"（5个 β_p）和"派生常数"（聚类中心 + δ 表），避免被质疑参数过多
5. 消融实验是论文区分度的关键——不只是报告"我们的模型最好"，而是展示**每个模块的独立贡献**
6. 若无法全部达标，诚实分析不可达标的原因比强行调参更有学术价值——例如"$u_{\text{nor},L}$ 受限于 D0 仅含1天10窗口，跨日泛化需要更多 D0 数据"
