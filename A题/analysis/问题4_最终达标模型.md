# 问题4：达标的最终模型 — 建模分析

## 一、问题分析

### 1.1 问题重述

综合问题1–3成果，建立满足五条工程要求的最终流量估计模型：

| # | 工程要求 |
|---|---------|
| ① | 输出每个窗口的 `model_volume_m3` |
| ② | 规则和参数在测试前固定 |
| ③ | 不用 date、window_id、standard_volume 等非运行时信息 |
| ④ | 数据有效性门须有物理含义 |
| ⑤ | 报告 MAE、组通过率、$u_{\text{nor},L}$、$u_{\text{nor},r}$、$u_{\text{nor},d}$ |

### 1.2 达标目标

$$|\bar{e}_g| \le 0.2\%,\quad SD_g \le 0.040\%,\quad u_{\text{nor},L} < 0.036\%,\quad u_{\text{nor},r} < 0.040\%,\quad u_{\text{nor},d} < 0.115\%$$

### 1.3 设计原则

1. **级联架构**：问题1基础积分 → 问题2流量点修正 → 问题3扰流补偿
2. **物理优先**：每一级修正有明确的物理对应，避免纯数据驱动
3. **参数最少化**：总共不超过 20 个可调参数，全部由 D0 + D1–D8 数据离线估计
4. **开关可控**：每一级可独立启用/关闭，便于误差溯源

## 二、最终模型架构

### 2.1 总流程图

```
输入：窗口在线特征
  chord0-4, profile_*, dyn_*, zero_*, flow_point
│
├─(1) 数据有效性门 ───────────────────────────
│   检查：dyn_active_eq_s ≥ τ_active
│         dyn_plateau_cv ≤ τ_cv
│   不通过 → 标记为无效窗口，不输出
│
├─(2) 基线估计 ──────────────────────────────
│   V_base = A · T · Σ(w_j^Lagrange · chord_j)
│   (来自问题1最优方法)
│
├─(3) 流量点修正 ────────────────────────────
│   V_1 = V_base · (1 + β_p)
│   β_p 由 D0 查表，5 个固定值
│   (来自问题2模型一)
│
├─(4) 扰流检测 ──────────────────────────────
│   |profile_ab_abs| > τ₁ or |profile_swirl| > τ₂
│   无扰流 → 跳到步骤(6)
│   有扰流 → 继续
│   (来自问题3子问题3)
│
├─(5) 扰流分类 + 补偿 ───────────────────────
│   扰流类型 c ← k-NN(聚类中心)
│   V_2 = V_1 · (1 + δ_p^(c))
│   δ_p^(c) 查表：类型 c × 流量点 p
│   (来自问题3子问题2+4)
│
├─(6) 输出 ──────────────────────────────────
│   model_volume_m3 = V_final
│
```

### 2.2 模块公式

#### (1) 数据有效性门

**物理含义**：窗口必须包含足够的稳定积分时长，且平台段波动受控。

| 门条件 | 物理含义 | 阈值 |
|--------|---------|------|
| `dyn_active_eq_s ≥ τ_active` | 窗口有效稳定时长足够 | $\tau_{\text{active}} = 30\text{s}$（窗口最短约116s，取 25%） |
| `dyn_plateau_cv ≤ τ_cv` | 平台段流速变异系数可控 | $\tau_{\text{cv}} = 0.10$（10%，工程经验值） |

不通过的窗口不输出 `model_volume_m3`，在提交 CSV 中可标记为 NaN 或移除该行。

#### (2) 基线积分

$$V_{\text{base}} = A \cdot T \cdot \sum_{j=0}^{4} w_j^{(L)} \cdot c_j$$

其中：
- $A = 0.13138$ m²（附件7等效截面积）
- $T$ = `duration_s`
- $w_j^{(L)}$ = Lagrange 型积分权重（附件7，$\sum w_j^{(L)} = 1$）
- $c_j$ = `chord0`–`chord4`

选择 Lagrange 而非 Phys6 的理由：问题1分析表明 Lagrange 均值误差最接近零（-0.008% vs Phys6 的 -0.136%），更适合作为无偏基线。

#### (3) 流量点修正

$$\beta_p = \frac{1}{N_p^{(0)}}\sum_{i \in \Omega_p^{(0)}}\frac{V_i^{\text{std}} - V_i^{\text{base}}}{V_i^{\text{base}}},\quad p \in \{40,50,60,70,80\}$$

对 D0 未覆盖的流量点 20、30：用线性外推

$$\beta_p = \beta_{40} + \frac{p - 40}{80 - 40} \cdot (\beta_{80} - \beta_{40}),\quad p \in \{20,30\}$$

#### (4) 扰流检测

$$\text{flag} = \mathbb{1}\left[|\text{profile\_ab\_abs}| > \tau_1 \text{ OR } |\text{profile\_swirl}| > \tau_2\right]$$

$$\tau_1 = 0.035,\quad \tau_2 = 0.021 \quad \text{(D0 max + 3σ)}$$

**物理可解释性**：`profile_ab_abs` 量化声道双向不对称——扰流件必然破坏对称性；`profile_swirl` 量化截面旋转——扰流件（弯头、旋流片）引入角动量。

#### (5) 扰流补偿

以 k=3 近邻到聚类中心（离线确定的 2-4 类），查表补偿：

| 扰流类型 | FP=20 | FP=30 | FP=40 | FP=50 | FP=60 | FP=70 | FP=80 |
|---------|-------|-------|-------|-------|-------|-------|-------|
| 类型A | δ₂₀ᴬ | δ₃₀ᴬ | δ₄₀ᴬ | δ₅₀ᴬ | δ₆₀ᴬ | δ₇₀ᴬ | δ₈₀ᴬ |
| 类型B | δ₂₀ᴮ | δ₃₀ᴮ | δ₄₀ᴮ | δ₅₀ᴮ | δ₆₀ᴮ | δ₇₀ᴮ | δ₈₀ᴮ |
| … | … | … | … | … | … | … | … |

$$\hat{V}_i = V_i^{\text{base}} \cdot (1 + \beta_{p_i} + \delta_{p_i}^{(c_i)})$$

## 三、离线参数估计

### 3.1 参数总表

| 参数 | 数量 | 来源 | 估计方式 |
|------|------|------|---------|
| $w_j^{(L)}$ | 5 | 附件7 | 固定 |
| $A$ | 1 | 附件7 | 固定 |
| $\beta_p$ | 5 | D0 | 直接计算 |
| $\tau_1, \tau_2$ | 2 | D0 | max + 3σ |
| $\tau_{\text{active}}, \tau_{\text{cv}}$ | 2 | 工程经验 | D0 分位数 |
| 聚类中心 | 4类×10维 | D1–D8 | 层次聚类+Ward |
| $\delta_p^{(c)}$ | ≤4类×7流量点 | D1–D8 | 组均值偏差 |
| **合计** | **≤47** | | **全部固定** |

### 3.2 防过拟合措施

1. 所有参数按组（日期/扰流/流量点）聚合后估计，不对单个窗口调优
2. $\beta_p$ 仅从 D0 (10窗口) 估计，不在 D1–D8 上重估
3. 聚类数由轮廓系数决定，上限 4 类
4. 交叉验证：对 8 天扰流数据做留一日交叉验证评估泛化

## 四、评价指标计算

### 4.1 全窗口指标

$$\text{MAE} = \frac{1}{N}\sum_{i=1}^{N} |e_i|,\quad e_i = \frac{\hat{V}_i - V_i^{\text{std}}}{V_i^{\text{std}}} \times 100\%$$

### 4.2 分组指标

对同日同流量点且 $n_g \ge 3$ 的组：

$$\bar{e}_g = \frac{1}{n_g}\sum_{k=1}^{n_g} e_{g,k},\quad SD_g = \sqrt{\frac{1}{n_g-1}\sum_{k=1}^{n_g} (e_{g,k} - \bar{e}_g)^2}$$

组达标：$|\bar{e}_g| \le 0.2\%$ 且 $SD_g \le 0.040\%$。

### 4.3 三项综合指标

**线性度等效量**（仅 D0，流量点 40–100）：

$$u_{\text{nor},L} = \sqrt{\frac{1}{m-1}\sum_{p=1}^{m}(\bar{e}_p^{(0)})^2}$$

**重复性等效量**：

$$u_{\text{nor},r} = \max_g SD_g$$

**扰流综合指标**：

$$u_{\text{nor},d} = \sqrt{u_{\text{nor},d,c}^2 + u_{\text{nor},d,r}^2}$$

其中：

$$u_{\text{nor},d,c} = \frac{\max_{r,p}|\Delta e_{r,p}|}{\sqrt{3}},\quad u_{\text{nor},d,r} = \max_{r,p} SD_{r,p}^{(d)}$$

## 五、伪代码

```python
# === 离线阶段：参数估计（仅运行一次） ===

# 加载数据
df = load_data()

# 1. 基线积分
V_base = area * duration * sum(w_lagrange[j] * chord[j] for j in 0..4)

# 2. 流量点修正 β_p
D0 = df[df.condition == 'no_disturbance']
for p in [40,50,60,70,80]:
    beta[p] = mean((V_std - V_base) / V_base) over D0 where flow_point == p
# 外推20,30
beta[20] = extrapolate(beta, 20)
beta[30] = extrapolate(beta, 30)

# 3. 扰流检测阈值
tau_1 = D0['profile_ab_abs'].abs().max() + 3 * D0['profile_ab_abs'].std()
tau_2 = D0['profile_swirl'].abs().max() + 3 * D0['profile_swirl'].std()

# 4. 扰流聚类（离线，可用disturbance_id）
features = [norm_chord_0..4, ab_0..4, profile_ab_abs, profile_swirl]
cluster_model = HierarchicalClustering(features[disturbed], n=2..4)

# 5. 扰流补偿表 δ_p^(c)
for each cluster c:
    for each flow_point p:
        delta[c][p] = mean(error over cluster c, flow p) - beta[p]


# === 在线阶段：窗口级推理（固定参数，逐窗口执行） ===

def predict(window):
    # 有效性门
    if window.dyn_active_eq_s < 30 or window.dyn_plateau_cv > 0.10:
        return None  # 无效窗口

    # 基线
    V = area * window.duration_s * sum(w[j] * window.chord[j])

    # 流量点修正
    V *= (1 + beta[window.flow_point])

    # 扰流检测
    is_dist = (abs(window.profile_ab_abs) > 0.035 or
               abs(window.profile_swirl) > 0.021)

    if is_dist:
        # 扰流分类
        x = extract_features(window)  # 10-dim
        c = cluster_model.predict(x)  # k-NN to nearest center
        # 扰流补偿
        V *= (1 + delta[c][window.flow_point])

    return V
```

## 八、注意事项

1. 最终模型 **不得** 使用 `standard_volume_m3`、`date`、`window_id` 作为输入——仅用于离线参数估计和评价
2. 所有阈值和参数必须在测试前固定——提交时附带 `params.json`
3. 有效性门应在论文中给出完整的物理推导——参考 ISO 12242:2012
4. 若部分扰流类型数据极少（如 D13 仅含 13 窗口），补偿参数可能不可靠——需汇报不确定性
5. 消融实验建议：逐步移除模块（无修正 → +流量点修正 → +扰流检测 → +扰流补偿），展示各模块贡献
6. 最终模型的所有公式、流程图和伪代码需与代码实现一一对应
