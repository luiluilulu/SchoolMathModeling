# 代码与写作规范

本文档为三人协作约定，覆盖 Python 代码、LaTeX 论文和注释风格。

---

## 一、注释原则

### 1.1 写什么

- 每个函数一行 docstring，说明输入/输出
- 非显而易见的数学公式对应的代码行，注释公式编号
- 关键参数取值的原因（如 `tol=1e-6  # 题目要求精度 10^-5`）

### 1.2 不写什么

- 不写"加载数据""数据预处理"等代码本身已经说清楚的事
- 不写废话注释：`x = x + 1  # 将 x 加 1`
- 不写过度解释：为什么用这个方法、方法原理（那是论文干的事）

### 1.3 禁止的 AI 味

| 禁止 | 替代 |
|------|------|
| `# 🚀 核心步骤：模型求解` | `# 模型求解` |
| `# 这里我们巧妙地利用...` | 不写，或只写公式编号 |
| `# 值得注意的是...` | 直接陈述 |
| `# 首先...其次...最后...` | 分三行短注释 |
| `# 该函数实现了强大的...` | `# 线性规划求解，返回 (x_opt, f_val)` |

## 二、Python 代码

### 2.1 文件头

```python
"""问题X：<一句话描述>。

求解方法：<算法名>。
输出：results/X_results.csv, figures/X_figure.png
"""
import numpy as np
import pandas as pd
```

三行搞定，不写作者、日期、长段说明。

### 2.2 命名

| 类型 | 风格 | 示例 |
|------|------|------|
| 变量/函数 | snake_case | `flow_rate`, `calc_volume()` |
| 常量 | UPPER_SNAKE | `N_CHANNELS = 5` |
| 类名 | PascalCase | `UltrasonicModel` |
| 私有函数 | `_` 前缀 | `_validate_input()` |

### 2.3 函数风格

```python
def calc_flow_volume(window_data, geometry):
    """窗口流量估计，返回体积估计值 (m^3).
    
    Args:
        window_data: pd.DataFrame, 窗口级数据
        geometry: pd.DataFrame, 几何参数
    
    Returns:
        np.ndarray, shape (n_windows,)
    """
    ...
```

- 小函数（≤30行），单一职责
- 避免全局变量，用参数传递
- 相对路径读取数据，`problem/` 下文件只读

### 2.4 变量命名

- 数学符号直译：`lambda_reg` 而非 `regularization_strength`
- 物理量带单位：`time_s`, `flow_m3h`, `length_mm`
- 不用拼音，不用无意义缩写

## 三、LaTeX 论文

### 3.1 项目约定

- 编译器：latexmk (xelatex)
- 主文件：`paper/main.tex`
- 章节拆分：`sections/01_xxx.tex` ~ `05_xxx.tex`
- 图：`paper/figures/`，引用相对路径
- 参考文献：`paper/ref.bib`

### 3.2 写作用语

**禁止的 AI 高频词**（来自技能"去AI味指南"）：

| 禁止 | 替代 |
|------|------|
| 此外、另外 | 直接陈述或"同时" |
| 深入探讨、充分展示 | 分析、表明 |
| 具有重要意义 | 不写，或具体说明 |
| 为...奠定基础 | 直接说明贡献 |
| 关键的、核心的 | 具体说明作用 |

**禁止句式**：
- "不仅...而且..."
- "从X到Y，从A到B"（虚假范围）
- "标志着...反映了...体现了..."
- 通用积极结论："前景广阔""迈出了重要一步"

**正确写法**：
```
该模型在测试集上准确率 95.6%，求解时间 0.3 秒。
```
而非：
```
该模型取得了令人振奋的结果，充分证明了方法的有效性和优越性。
```

### 3.3 禁止

- emoji 表情符号
- 破折号过度使用（——）
- 粗体强调（用数据说话，不用格式强调）
- "专家认为""观察者指出"等模糊归因

- 不要提交 `paper/*.aux` `*.log` `*.out` `__pycache__/`（已在 .gitignore）
- 大结果文件（>1MB）不追踪，在 output/results/ 下 .gitignore 已忽略

- 不提交 `.ipynb_checkpoints/`
